# План прототипа — Volga Dream (турагенту на теплоходе)

## Идея

Приложение для туриста, плывущего на теплоходе по туристическому маршруту. Экскурсия на несколько дней с программой: лекции, экскурсии, приёмы пищи. Нужен просмотр расписания на каждый день и пуш-уведомления перед мероприятиями.

## Core-экраны (минимум для прототипа)

1. **Главная — расписание на сегодня** (список активностей по времени)
2. **Детальный экран активности** (описание, время, место)
3. **Расписание на выбранный день** (свайп или вкладки по дням)
4. **Экран уведомлений/запланированного** (опционально, заглушка)

## Модели данных

- `Activity` — id, title, description, type (lecture/excursion/meal), startTime, endTime, location, dayIndex
- `DayItinerary` — dayIndex, date, title, List\<Activity\>
- `Cruise` — name, shipName, List\<DayItinerary\>

## Сервисы (мок)

- `MockCruiseService` — возвращает заранее подготовленные данные по дням
- `NotificationService` — имитация планирования уведомлений (без реального push, пока просто print или snackbar)

## UI

- Material Design 3 (Material You), навигация BottomNavigationBar или TabBar по дням
- Тема: сине-голубая (ассоциация с рекой/Волгой), название Volga Dream

## Пошаговый план

1. Создать модели данных (`lib/models/cruise.dart`)
2. Написать мок-сервис (`lib/services/cruise_service.dart`)
3. Главный экран: лента активностей на сегодня
4. TabBar: переключение по дням
5. Детальный экран активности (onTap)
6. Экран с информацией о круизе (app bar menu)
7. Прототип != production — без авторизации, анимаций, обработки ошибок

## Когда разделять агентов

Когда проект вырастет до продакшена:
- **Дизайнер** — верстает макеты (Figma)
- **iOS-агент** — адаптирует под Cupertino
- **Android-агент** — адаптирует под Material

На стадии прототипа один агент + правки пользователя — оптимально.
